Fingerprints

By: Will Turner



A secure HTTPS connection is one of the most important things on the internet. It ensures your security and privacy while browsing the internet. Although HTTPS is secure there is something know as an HTTPS proxy appliance which is a deviice that circumvents the encryption of HTTPS by adding a pseudo certificate authority to the users device after they connect to a network. This can pose a threat of you being attacked by an MITM (man in the middle) attack which is where security is breached partway through the connection between your device and the server and its used to infect your device. One way this can be stopped is through creating a hash which is a complex mathematical algorith which carefully processes every bit of what they "digest". originally MD5 was the hashing algorithm used by all of the internet but it was found to be a little weak so it was replaced by SHA1. A strong or weak hash is determined by how much the fingerprint is changed if just 1 character of the hash is changed. A certificate authority is an entity that proves their identity through documentation, telephone numbers, etc. When an internet browser establishes a secure coonnection that site must provide a signed CA for the web browsers inspection. SSL Interception cannot be prevented but can be reliably detected because it is not possible to completely spoof any security certificate. A false positive can usually occur with bigger websites that contain multiple CAs which can lead people to have a fingerprint mismatch. A false negative is whenever a website tests negative but really is positive.
